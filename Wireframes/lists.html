Invariably, list content is static. There are however instances when rows will use a related in-line action. These should adhere to list layout guidelines.

 

Where required (or possible), data should be easy to filter or sort. Observe the data tagging methods by which it is currently possible to filter and provide tools accordingly. For example, radio buttons, check boxes and sliders.

 

'Sort by' method is by droplist menu containing relevant and available sorting methods.

BEHAVIOUR

A consistent use of the white canvas, type, keylines and proximity add clarity to the information, ensuring the display is easy to understand. Lists mostly consist of short data that fits into a single column. However, in cases where a column line or data content is longer, observe the text wrap convention.

 

There are no fixed column widths, though be mindful of how many are employed. Too many columns will become cumbersome and hard to read. Define and adjust column widths as required.
