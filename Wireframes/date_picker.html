When a user wants to find or submit information based on a date or date range, a date picker is required. It is a familiar graphical interface that is commonly understood among users.

WHAT IS IT FOR?

DATE PICKER

IHTSDO

INTERACTION

Do not use when the date to be selected is more easily inputted via writing the date as text. An example could be date of birth, where many clicks back in years would be required.

 

If a date range selection is required, never let the end-date be before the start date. That means listening to the start-date for changes and changing the end-date if the start date is set to anything bigger.

 

Disable any days that are not valid as part of a selection.

CONSIDERATIONS

On activation, a box with a month-calendar is laid over the current page, prompting the user to select a date in the box. Left and right arrows allow user to cycle through months.

 

To give a sense of life, selection should default on today's date. A rollover state feeds back user's interaction, until they make their selection.

 

The form field should allow users to type a date, which in turn quick links to it accordingly.

 

On-selection of a date, the form field will be populated with that date. User can select again if required.

BEHAVIOUR

Use when there is a requirement to choose a date or date range in order to submit, track, sort, or filter data.
