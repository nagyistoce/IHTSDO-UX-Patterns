Forms allow a user to enter information that is then sent as data to a server for processing. Examples are: Feedback and sign up.

INTERACTION

In order to help the user achieve a goal, forms need to be usable. Forms are clean and uncomplicated. Fields are accompanied by an instructional label (one word or VERY short term) and in-field hint text where appropriate. To help focus a user's view, each currently selected form field is enhanced by a highlight.

 

Utilise autocomplete where possible. e.g. country selection droplist (see droplist guidelines).

 

To improve user experience, consider these points:

 

If a user needs to go back to fix a mistake that they have made within the form, where appropriate, their information should still be there so that they don’t need to re-enter information.

 

Forms should know when to move to the next field. Provided the information is correct, the movement through a form should be fast and fluid.

 

Adhere to Accessibility Guidelines standards, by always ensuring the tab order is correct between any elements on screen.

BEHAVIOUR

An input text form field is presented with an accompanying label that describes what input is expected in the field.

 

In the context of forms, order the labels logically. For example, ask for name before any contact details or any other number of other questions. More involved questions should come towards the end of the form. Group related information, such as personal details. The flow from one set of questions to the next will be more logical.

 

The length of the form element should be determined by the maximum amount of characters a user could enter.

Disabled text field

In a case when field must be disabled

 

 

 

Clearly distinguish which input fields cannot be left blank by the user. Use of the conventional, mandatory field asterisk '*' is employed.

 

Do not place form fields side by side in columns. Users often don't notice the column on the right.

 

Only use one field for telephone numbers (i.e. no area code box). Many users will try and put the full number in the area code box.

 

Avoid long, complex explanations at the top of forms. Users rarely read them. Forms should be simple enough to not need it anyway.

 

Enable OS specific shortcuts in-field, such as select all.
