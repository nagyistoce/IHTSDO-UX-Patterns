Allows users to quickly access information among heavy content. When browsing is not quick enough, 'Search' is the most efficient means for a user to find exactly what they require.

WHAT IS IT FOR?

SEARCH

IHTSDO

INTERACTION

Search puts users in control of how they find information, so there are pre-selectable parameters to maintain familiar search methods.

 

The search box is a clearly visible and transportable module across the IHTSDO services, so is quickly recognisable and easy to use.

 

On-focus of the search field, a droplist immediately displays any previously searched terms as a history. This feature caches a maximum of 15 terms and is scrollable as a list. Users can also remove history terms individually using the in-line delete function or clear the entire list on-click of the 'Clear all' option in the drop list. In a case when there is no search history, no droplist is displayed.

 

Search employs type-ahead functionality, meaning that when a user has begun entering their search term, it will start presenting matching items in a drop list, even though the user is still entering their query. This feature is invoked only after three characters have been entered. As the user types in more text to the text field, matching items are narrowed down as the query gets more specific.

 

The search action button label should read 'Search'. Enable keystroke 'Return' to perform this action as users are familiar and expect it. It's also possible to clear the search term in the field using the in-line 'X'.

 

Submitting a search term, returns a result set as appropriate:

 

• All results featuring searched term

 

• Exact result

 

• No results found. In this case, suggestions, popular searches and the ability to search again should be offered to the user

 

The search results page is the prime focus of the search experience.

BEHAVIOUR

The search field should be the prime focal feature of a page when a user is at the start of a search journey. Where appropriate and possible, it should be accompanied by the option to instead browse. At search results, it should be easily accessible should a user wish to perform another search.
