Small basic droplist + Label

Avoid very long menus that require extended scrolling. This makes it extremely difficult for users to see all of their choices in one glance. In cases like this, it's advisable to present such long lists of options as a regular HTML list of hypertext links.

 

Never create a droplist menu with more than one level. The menus are very difficult to use, because if the user even slightly loses focus of the menu with the mouse pointer, they have to start from the top.

CONSIDERATIONS

A droplist menu is activated by clicking it's corresponding button, then deactivated by clicking the button once more. On-click, the droplist menu should appear seamlessly and without interruption.

 

The usability of the navigation within the menu itself is enhanced further by a hover effect. This shows exactly which button in the navigation menu the user will select.

BEHAVIOUR

Droplists are displayed with an on-button label that describes their content. However, occasionaly, a preceding label may be required. Such as part of a form with a label 'Select Title'.

 

Droplists should contain more than 2 elements and a maximum of 15. If a list contains more than eight elements, the most important or frequently used elements should be displayed as a group at the top of the list. This can be clearly defined by a thin line to separate the list.

 

Content is typically text, but in some cases can be accompanied by an icon. Such as with a Language selection.

 

As part of form selection, droplists can prevent users from entering erroneous data, since they only show legal choices. They can be used to make a preference selection, such as a 'Sort by' or language. These examples require no accompaniment of action button and would change a view according to selection.

 

Some droplists should be accompanied by an action button, which allows a user to make a selection, but change their mind before submitting. In this sense, it's a 'non-destructive selection'. Consider when this is appropriate.
