On single nest modules (4 columns), a rollover feeds back that the module is clickable. On-click, links to corresponding content. The exception to this case is when a list of links are nested within the module. In this case, there is no rollover state of the module, but the hyperlinks behave as defined. Secondary, support actions nested on the module link according to their function.

 

Large nest modules (12 columns) hold multiple sets of primary and secondary actions and static content. In this case, actions are 'hit' directly as required.

BEHAVIOUR

Nest modules are typically used as a cluster on the first page of a service or dashboard. Consider the target user as to which module is most appropriate. For example, a clinician may only need to know the service type they wish to link to, where as a developer may need individual links to a code resource (such as GitHub).

 

Single nest modules can be likened to a 'folder' that contains a set of information. Large nest modules can be likened to the filing case in-which the folders sit. Large modules are an overview of all folders.
