TABS

IHTSDO

INTERACTION

Navigation tabs derive from the physical metaphor of folders in a file-cabinet and are consequently familiar to the user.

 

Module tabs provide an easy way to show large amounts of similar structured or related data, parted by categories and displayed in different panes. Each pane is viewable one at a time.

WHAT ARE THEY FOR?

A default tab will always display. On-click of subsequent tabs, content in the container associated to it is displayed. Tab buttons have three states: selected, rollover and deselected. They do not link out, so keep a user in page.

BEHAVIOUR

The primary goal of the tabs UI pattern, is to permit users to view a group of related data one at a time, saving valuable screen real estate. With this in mind, it is paramount to observe that tabs must only be used with content that has some connection to each other, so users can see a logical correlation.

 

They should be used in a case when a large set of content is required to be organised and displayed on a single page, without sacrificing the amount of information displayed at once.

HOW AND WHEN TO USE

EXAMPLES

Page Tab Control

This module is used when related content on a page needs to be divided, but accessible in-page.

In-module Tab Control

Smaller in scale, this module is used when related content in a module needs to be divided, but accessible in-page. Example: SNOMED CT result of summary, details etc.
