This resource provides the guidelines and information required to create great and consistent IHTSDO interfaces. Each page is a pattern-type and provides written description overviews in consistent sections:

'What is this?'; 'Behaviour'; 'How and when to use', 'Examples'; and in cases where necessary, 'Considerations'. There is a small table of Page Use examples, which identifies current IHTSDO pages the pattern might be used for, including some proposals.

 

Examples are also provided (frequently annotated). Wireframes are typically interactive to demonstrate the behaviour and in some cases it's possible to toggle the view between the wireframe and design.
